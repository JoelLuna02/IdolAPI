### **¿Qué es una API?**

_Una API REST (también conocida como API RESTful) es una interfaz de programación de aplicaciones (API o API web)
que se ajusta a las limitaciones del estilo arquitectónico REST y permite la interacción con los servicios web RESTful.
REST significa transferencia de estado representacional y fue creado por el informático Roy Fielding._

Una API es un conjunto de definiciones y protocolos para crear e integrar software de aplicaciones. A veces se lo denomina
contrato entre un proveedor de información y un usuario de información: se establece el contenido requerido por el consumidor
(la llamada) y el contenido requerido por el productor (la respuesta). Por ejemplo, el diseño de API para un servicio
meteorológico podría especificar que el usuario proporcione un código postal y que el productor responda con una respuesta
de dos partes, siendo la primera la temperatura alta y la segunda la temperatura baja.

En otras palabras, si desea interactuar con una computadora o sistema para recuperar información o realizar una función,
una API lo ayuda a comunicar lo que desea a ese sistema para que pueda comprender y cumplir con la solicitud.

[Más información aquí](https://www.redhat.com/es/topics/api/what-is-a-rest-api#%C2%BFqu%C3%A9-es-una-api-rest)

## **Pila tecnológica**

Este proyecto se desarrolla a través de node.js, utilizando Express.js como marco de desarrollo backend,
JWT para implementar la autenticación a través de credenciales, GraphQL para realizar consultas y mutaciones,
y Handlebars para el renderizado de templados html.

También cuenta con MongoDB, para almacenar información y bytes en una base de datos tipo NoSQL.

La base de datos de producción se despliega en MongoDB Atlas y la aplicación, desde Vercel.
